БД PostgreSQL, 4 таблицы:
1. card_user - для хранения данных о пользователе. 
В рамках данной задачи хранятся не только ФИО, но также креды (login/password), причем пароль хранится 
в зашифрованном виде (в моем случае кодирование в Bcrypt password encoder со сложностью 5). Также там хранится роль\
(есть две роли: ROLE_USER (умеет получать данные) и ROLE_ADMIN (умеет вносить изменения))
Такое решение не надо использовать на практике, оно было выбрано чисто для данного задания.
2. card - для хранения данных о карте пользователя. Принято решение, что у пользователя может быть одна карта.
3. bonus - для хранения АКТУАЛЬНЫХ данных о бонусах на карте. Сущность из этой таблицы связано с картой 1:1
4. bonus_history - историчекая таблица, где хранятся все операции с бонусами, соверщшенные по данной карте. 
С одной картой может быть связано множество записей из bonus_history. Максимально наивная реализация. 
При каждой операции изменения баланса бонусов на карте происходит апдейт записи в bonus и create записи в bonus_history.
Соответстенно, текущий баланс получаем из бонуса, выписку истории по карте - из bonus_history. Отдельная историческая
таблица заведена, чтобы не засорять актуальные данные и не замедлять выполнение операций изменения баланса (пик нужной записи, перестроение индексов и т.д.).
